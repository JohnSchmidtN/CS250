## How do I interpret user needs and implement them into a program? How does creating "user stories" help with this?

Throughout this course we talked about the importance of User Stories and how they can contribute to allowing the developers better access to the information needed for the task they are working on. I believe that User Stories fill a need in the Scrum-Agile methodology by using the format of "As a [Persona], I [Want To], [So That]" they clearly state what is needed for each task.

Before the start of the program I wasn't really using any Agile processes when developing programs. After taking this course my goal is to start implementing all of the Agile processes into my future development work. I would like to start by focusing on spending more time with designs and requirment gathering. ## What does it mean to be a good team member in software development?

To me, being a good team member in software development is making sure that you are offering open communication while working in a group setting. Participation is another way that you can be a good team member. Actively participating in meetings, daily scrums, or face-to-face can help to make sure that everyone on the team feels as if they are contributing.
